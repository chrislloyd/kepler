Story

Humanity is interstellar. Planets are starting to be colonized with the help of robot companions (“man's best friend”). As humans travel to the boundaries of the universe, they also push the boundaries of artificial intelligence. The game starts at the cusp of the singularity on the planet Kepler 1a. You are a robot trying to achieve sentience. Create your own destiny in an unforgiving environment.

High level overview

There's a central server. It's running on chrislloyd.local:5000. The server only speaks websockets. Your bot is going to connect to this server (via. websockets). Every 300ms or so, your client will recieve a tick. This data is formatted in JSON.

Stuff you get from the server


  {
    "event": "tick",
    "state": {
      "tick": "abcd-1234",
      "entity": "abcd-1234",
      "entities": [{
        "id": "abcd-1234",
        "pos": {"x": 0, "y": 0},
        ...
      }]
    }
  }

Cool! The tick is just a random ID that changes. For simple bots you can probably ignore it. The root entity (state/entity) is your bot's ID. The entities array is just a collection of other players and things in the game.

Commands

You can send commands back to the server via. the websocket. They need to be in JSON.

MOVE "up" | "down" | "left" | "right" Moves your bot
TURN Int Turns your bot around
SHOOT Goes boom
NAME String Set's your bot's name
FLAIR Int, Int, Int Set's your bot's flair color. 3 arguments, each is an integer between 0-255 representing r, g & b color components.

These command should be formatted as tuples and serialized as JSON.

["MOVE", "up"] ["NAME", "chrislloyd"] ["FLAIR", 255, 255, 255]
